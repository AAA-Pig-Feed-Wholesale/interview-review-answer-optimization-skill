## 它能做什么

这套 skill 用来处理已经发生过的技术面试，或者已有问答记录的模拟面试。它重点做三件事：

- 提取面试官到底问了什么，整理问题簇和追问链
- 判断每道题在考什么、你原回答的问题在哪里
- 在不虚构经历的前提下，给出更稳、更贴岗、更能落到口语表达中的答法

它不是：

- 简历包装器
- 项目编造器
- 脱离真实背景的万能话术库

## 适合谁用

当前版本优先面向技术岗，尤其适合：

- AI 应用开发
- RAG
- Agent
- 后端

如果岗位仍属于技术岗，但不完全落在上面这些方向，也可以使用，只是岗位特化判断会更保守。

## 它的输出是什么

默认会围绕“复盘”和“优化”产出这些内容：

| 模块 | 作用 |
| --- | --- |
| 候选人画像 | 先确定你的真实背景边界、优势方向和高风险项 |
| 问题提取 | 提取面试官问题、用户反问、追问链和问题簇 |
| 题型判断 | 判断是项目题、场景题、系统设计题、八股题、行为题还是 JD 匹配题 |
| 回答诊断 | 分析原回答为什么减分、哪里空、哪里不贴岗 |
| 回答优化 | 给出更好的回答思路、个性化回答、优秀回答 |
| 整场总结 | 汇总整场面试的主要短板与优先补强方向 |

## 输入什么

### 最低完整输入

- 简历
- 面试对话记录

### 可选增强输入

- 岗位 JD
- 项目材料 / 项目说明 / 项目总结
- 自我介绍稿
- 其他背景材料

### 输入越完整，提升主要体现在哪

| 补充材料 | 会增强什么 |
| --- | --- |
| JD | 岗位贴合分析、考点判断、“这题为什么问” 的识别准确度 |
| 项目材料 | 个性化回答质量、项目题细节支撑、真实素材调用能力 |
| 自我介绍稿 | 回答风格一致性、表达习惯对齐 |

如果缺少简历，则只能做通用复盘，不做强个性化回答映射。  
如果缺少面试记录，则不能进入逐题复盘，只能做准备建议。

## 怎么用

### 1. 选择输出模式

这套 skill 支持 3 种模式：

| 模式 | 适合什么场景 |
| --- | --- |
| `extract_only` | 先确认面试官到底问了什么、题型是什么、材料够不够 |
| `review` | 先看原回答的问题、每题考点、整场短板 |
| `optimize` | 直接获得逐题优化回答和更强的现场表达版本 |

默认模式是 `review`。

### 2. 最小使用方式

给出材料后，可以直接这样说：

```text
帮我先做 review
```

或者：

```text
帮我按 optimize 模式逐题优化
```

或者：

```text
先 extract_only，把问题提出来
```

### 3. 如果你想练重点题

你还可以继续要求：

- 保留原始追问链
- 对重点题补 30 秒版 / 90 秒版
- 对边界题补记忆锚点

## 这套 skill 的特点

### 先画像，再逐题

它不会一上来就逐题改答案，而是先根据简历、JD、项目材料等信息构建候选人画像，用来确定：

- 你的目标岗位方向
- 最相关项目经历
- 后续优先调用哪些真实素材
- 简历中的高风险项
- 更适合你的优势表达方向
- 回答风格建议

这样后续优化更像“基于你本人做回答优化”，而不是模板替换。

### 双层回答机制

很多题不会只给一种答案，而是区分两层：

- 个性化回答：基于你的真实经历来写，不虚构
- 优秀回答：按理想候选人标准给出更成熟的参考答法

对高价值题，还可以按需要补：

- 记忆锚点 / Golden Phrase
- 30 秒极简版
- 90 秒面试展开版
- 原始追问链
- 追问耐受度 / 当前成熟度

## 设计原则

这个 skill 的核心约束只有一句话：

> 可以优化表达、结构和顺序，但不能补造事实。

因此它会始终坚持：

- 不虚构经历、职责、指标、结果
- 不把弱相关素材硬写成强证据
- 不在材料不足时假装“高贴合个性化”
- 不把“知道概念”写成“做过落地”
