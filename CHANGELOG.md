# Changelog

## v1.3

### 本次升级重点

- 统一高价值题的增强粒度，减少同一份输出里“有些题很强、有些题偏旧风格”的不均匀问题。
- 强化记忆锚点 / Golden Phrase 的优先覆盖规则，优先服务高频主线和概念边界题。
- 强化“危险表达 / 为什么减分 / 替代说法”三段式，让点评更贴近真实口语失分点。
- 强化 30 秒极简版 / 90 秒面试展开版的适用边界，让输出更像可直接训练的面试实战稿。
- 强化业务分层案例输出，要求系统设计题、治理题、架构题更具体地体现分层职责、成本、延迟与可控性意识。
- 强化具像案例 / 证据绑定示例，鼓励对幻觉治理、规则问答、工具误调用、长期记忆取舍等题给出最小可口述案例。

### 规则层增强

- 为高价值题新增“最小增强集”规则：至少补齐 3 个训练增强字段，避免只保留旧式“问题 + 回答”结构。
- 新增统一性约束：同一份输出中，如果某条主线的高价值题已经启用增强模块，其他同层级、同主线题目也应尽量保持相似增强粒度。
- 在 `optimize` 模式下，默认优先补齐记忆锚点、30 秒版、90 秒版、危险表达三段式、追问训练标签等高价值字段。

## v1.2

### 新增能力

- 新增记忆锚点 / Golden Phrase，用于边界题、框架区分题和高频抽象题的临场记忆。
- 新增“问题簇 + 原始追问链”双层保留，既保留抽象能力，也保留真实追问压力。
- 将“原回答的问题”强化为三段式：危险表达 / 为什么减分 / 替代说法。
- 为高价值题增加 30 秒极简版、90 秒面试展开版、完整参考版（如有必要）。
- 对系统设计题、场景题、工程治理题增加业务分层案例输出。
- 对证据绑定、幻觉治理、检索到了但答错等问题，增加具像案例 / 证据绑定示例。
- 强化“可选追问及补充答法”，加入反打型、边界型、失败场景型追问。
- 增加“追问耐受度 / 当前成熟度”标签，方便训练排序。

### 保留能力

- 保留结构化复盘、候选人画像、问题簇抽象、考察点分析、双轨回答机制。
- 保留真实性优先、不虚构经历、输入不足时降级等边界控制。
- 保留三种输出模式：`extract_only` / `review` / `optimize`。

## v1.1

### 新增

在原有结构化复盘框架上，增强了模式切分和输出协议稳定性：

- 默认输出模式改为 `review`
- `optimize` 模式保留完整逐题优化能力

同时统一了输入降级规则、题型判断逻辑和输出字段。

### 主要增强点

1. 输出模式更加清晰
   - 明确区分 `extract_only`、`review`、`optimize`
   - 默认模式为 `review`
   - 模式切换规则更稳定

2. `extract_only` 能力更独立
   - 可以只做问题提取
   - 可以只做结构化整理
   - 适合在材料较乱时先抽出问题和分类

3. `optimize` 输出规范增强
   - 保留完整逐题输出字段
   - 保留个性化回答与优秀回答
   - 保留可选追问与补充答法

4. 统一输入降级规则
   - 缺少简历时只能做通用复盘
   - 缺少面试记录时不能做标准逐题复盘

5. 文档结构收口
   - README 只做导航
   - Prompt 模板独立
   - 示例文件独立
