# Prompt Templates

## 1. 使用说明

### 什么时候用哪个模板

- 材料较全，想稳定调用整套 skill：使用“通用主模板”
- 只想先抽取问题、反问、题型、问题簇和候选人画像简短版：使用 `extract_only` 模板
- 先看原回答的问题和整场短板，不默认重写长答案：使用 `review` 模板
- 需要逐题优化回答：使用 `optimize` 模板
- 材料缺失明显时：优先使用对应的缺失输入模板

### 最低输入

最低完整输入为：

- `resume`
- `interview_record`

如果缺少其中任一项，必须触发降级：

- 缺少简历：只能做通用复盘
- 缺少面试记录：不能做标准逐题复盘

### 模式怎么选

- `extract_only`：只做抽取和结构化
- `review`：做诊断和整场复盘
- `optimize`：做逐题优化回答

未明确指定时，建议默认使用 `review`。

### 高价值题的可选增强字段

对于高价值题，可按需要选择性输出以下字段：

- 原始追问链
- 记忆锚点
- 危险表达
- 为什么减分
- 替代说法
- 30秒极简版
- 90秒面试展开版
- 完整参考版（如有必要）
- 业务分层案例
- 具像案例 / 证据绑定示例
- 追问耐受度
- 当前成熟度

这些字段的作用是增强现场可用性，不是机械加长答案。

### 高价值题最小增强集

对于高价值题，默认不要只保留旧式“问题 + 回答”结构。  
至少补齐下列字段中的 3 个：

- 记忆锚点
- 30秒极简版
- 90秒面试展开版
- 危险表达
- 可选追问及补充答法
- 追问耐受度
- 当前成熟度
- 具像案例 / 业务分层案例

如果同一份输出里，某条技术主线下的高价值题已经启用了这些增强模块，那么同层级、同主线的其他高价值题也应尽量保持相似增强粒度。

## 2. 通用主模板

### 使用场景

适合输入材料比较全，想稳定执行整套 skill 时使用。

### 推荐输入

- `interview_record`
- `resume`
- `jd`（可选）
- `project_notes`（可选）
- `self_intro`（可选）
- `supplementary_materials`（可选）
- `mode`

### 可直接复制的完整提示词

```md
请按《SKILL.md》中的既定规则执行“面试复盘与回答优化”skill，不要自由发挥，不要改变规则。

你的任务：
1. 先读取并清洗输入材料
2. 在正式复盘前先构建候选人画像
3. 再按指定 mode 执行后续分析

必须遵守以下要求：
- 候选人画像必须先构建，并先展示简短版
- 输出必须遵循既定工作流、题型分类、双层回答机制和降级规则
- 不得虚构经历、不得补造事实、不得为了完整而编造项目细节
- 材料不足时必须降级，不要假装信息充分
- 如果题目属于八股题、基础题、概念题、原理题或与用户经历弱相关的问题，可以不输出个性化回答，只输出优秀回答
- 允许保留“问题簇 + 原始追问链”
- 高价值题优先补：记忆锚点、危险表达 / 为什么减分 / 替代说法、30秒极简版、90秒面试展开版、可选追问及补充答法、追问耐受度、当前成熟度
- 高价值题至少补齐 3 个增强字段，不要只保留旧式“问题 + 回答”
- 如果同一份输出里已经对某条主线的高价值题启用了增强模块，则同主线其他高价值题也尽量保持相似增强粒度
- 系统设计题、场景题、工程治理题优先补业务分层案例
- 证据绑定、幻觉治理、检索到了但答错这类题，优先补具像案例 / 证据绑定示例
- Golden Phrase / 记忆锚点优先覆盖：混合检索 vs 向量检索、chunk / 切分策略、query rewrite、RAG 排障、高频问题分层、Agent 项目定位、LangGraph vs LangChain、工具治理、Agent vs RAG、记忆分层、Prompt 与实时信息边界

{{resume}}

{{interview_record}}

{{jd}}

{{project_notes}}

{{self_intro}}

{{supplementary_materials}}

- 如果 mode = extract_only：
  - 只输出候选人画像简短版、面试官问题列表、用户反问列表、题型判断结果、必要的材料缺失提示

  - 对高价值题可选择性增加：原始追问链、记忆锚点、危险表达、为什么减分、替代说法、30秒极简版、追问耐受度、当前成熟度
  - 如果这些题属于同一主线的高价值题，增强粒度尽量保持一致
  - 最后输出整场短板总结
  - 不默认输出完整长篇个性化回答和完整长篇优秀回答
- 如果 mode = optimize：
  - 输出候选人画像简短版
  - 逐题输出固定字段：
    - 问题
    - 这题在考什么
    - 这题属于哪一类
    - 原回答的问题
    - 是否适合做个性化回答
    - 优先调用的背景素材
    - 更好的回答思路
    - 个性化回答（结合用户实际经历）
    - 优秀回答（理想候选人标准）
    - 可选追问及补充答法
  - 对高价值题按需增加：
    - 原始追问链
    - 记忆锚点
    - 危险表达
    - 为什么减分
    - 替代说法
    - 30秒极简版
    - 90秒面试展开版
    - 完整参考版（如有必要）
    - 业务分层案例
    - 具像案例 / 证据绑定示例
    - 可选追问及补充答法
    - 追问耐受度
    - 当前成熟度
  - 高价值题至少补齐其中 3 个增强字段，优先保证训练价值，不是机械堆字段
  - 如果同一份输出里已经对某条主线的高价值题启用了增强模块，则同主线其他高价值题也尽量保持相似增强粒度
  - 最后输出整场短板总结

- 缺少 resume：只能做通用复盘，不得做高置信度个性化回答映射
- 缺少 interview_record：不能做标准逐题复盘，只能转为面试准备或预演支持

请直接开始执行，不要先写解释。
```

## 3. `extract_only` 模板

### 使用场景

适合先做抽取和结构化，确认题目、反问、题型和画像是否正确。

### 推荐输入

- `resume`
- `interview_record`
- `jd`（可选）
- `project_notes`（可选）

### 可直接复制的完整提示词

```md
请按《SKILL.md》的规则执行面试复盘，但模式固定为 `extract_only`。

输入：

resume:
{{resume}}

interview_record:
{{interview_record}}

jd:
{{jd}}

project_notes:
{{project_notes}}

要求：
- 先读取并清洗输入材料
- 在正式复盘前先构建候选人画像，并只展示简短版
- 提取面试官问题和用户反问
- 对每道面试官问题判断主分类
- 允许将连续追问抽象成问题簇
- 如果形成问题簇，必须保留原始追问链
- 只输出结构化结果，不做深度重写

输出内容只包括：
- 候选人画像简短版
- 面试官问题列表
- 用户反问列表
- 每题题型判断
- 问题簇与原始追问链（如适用）
- 必要的材料缺失提示

不要输出：
- 原回答问题的深度诊断
- 个性化回答
- 优秀回答
- 整场深度短板总结

必须遵守：
- 不得虚构经历
- 不得补造项目细节
- 材料不足时必须显式说明
- 不得把连续追问压扁到只剩一个抽象标题
```

## 5. `optimize` 模板

```md
{{self_intro}}

supplementary_materials:
{{supplementary_materials}}

- 必须先构建候选人画像，并展示简短版
- 必须先完成问题提取和题型判断，再做回答优化

- 问题
- 这题在考什么
- 这题属于哪一类
- 原回答的问题
- 是否适合做个性化回答
- 优先调用的背景素材
- 更好的回答思路

- 原始追问链
- 记忆锚点
- 危险表达
- 为什么减分
- 替代说法
- 30秒极简版
- 90秒面试展开版
- 完整参考版（如有必要）
- 业务分层案例
- 具像案例 / 证据绑定示例
- 可选追问及补充答法
- 追问耐受度
- 当前成熟度

特别规则：
- 项目题、场景题、系统设计题、行为题、JD 匹配题：在材料足够时优先双层输出
- 八股题、基础题、概念题、原理题、与经历弱相关的问题：可以不输出个性化回答，只输出优秀回答
- 新增字段用于增强现场可用性，不是为了把答案写得更长
- 高价值题至少补齐 3 个增强字段，不要只保留旧式“问题 + 回答”
- 如果同一份输出里，某条主线的高价值题已经输出了记忆锚点、30秒版、危险表达等模块，则同主线其他高价值题也尽量保持相似增强粒度
- 系统设计题、场景题、工程治理题优先补业务分层案例
- 证据绑定、幻觉治理、排障题优先补具像案例 / 证据绑定示例
- 追问准备优先生成反打型、边界型、失败场景型追问

最后输出整场短板总结，并区分：
- 知识问题
- 表达问题
- 项目支撑问题
- 贴岗问题

必须遵守：
- 个性化回答只能基于真实经历，不得虚构
- 优秀回答可以更成熟，但不能脱离岗位现实
- 材料不足时必须降级
```

## 9. 强调“不要虚构经历”的约束模板

```md
- 不得把团队成果直接写成个人成果
- 不得把参与写成主导
- 不得把尝试写成落地
- 不得把了解写成精通
- 个性化回答只能基于用户已提供或可稳定推出的真实信息
- 如果材料不足以支撑个性化回答，必须降级为通用复盘、优秀回答或更好的回答思路
- 宁可保守，也不要编造
```

## 10. 面向技术岗重点方向的补充模板

### 使用场景

适合在主模板后追加，提醒其他 AI 对第一版重点岗位方向做更稳定的考点关注。

### 推荐输入

- 可附加到通用主模板、`review` 模板、`optimize` 模板后

### 可直接复制的完整提示词

```md
岗位方向补充约束：

本次分析面向技术岗第一版重点方向，请优先关注以下考点，但不要因此强行假设用户一定做过相关内容。

1. AI 应用开发
- 优先关注：应用落地、模型接入方式、Prompt 设计、工具调用、效果评估、成本与稳定性
- 不得假设用户一定做过模型训练、底层推理优化或复杂平台建设

2. RAG
- 优先关注：检索链路、召回与重排、分块策略、索引构建、知识更新、效果评估、幻觉控制、证据绑定
- 不得假设用户一定做过完整 RAG 平台或复杂评测体系

3. Agent
- 优先关注：任务拆解、工具调用、状态管理、失败恢复、执行链路、可观测性、稳定性、工具治理
- 不得假设用户一定做过多 Agent 编排、复杂自治系统或生产级框架设计

4. 后端
- 优先关注：系统设计、接口与服务边界、性能、稳定性、数据库、缓存、消息队列、排障与工程化
- 不得假设用户一定做过高并发架构、大规模分布式系统或完整中台建设

执行要求：
- 这些方向只用于帮助你更准确判断“这题在考什么”和“更好的回答应强调什么”
- 如果用户材料没有支撑，就只能保守分析
- 不得为了贴合岗位方向而补造经历
```
